# Documentation style

I prefer markdown in docstrings. Yes, it's nonstandard, but all the internal docs and other text is markdown, and I don't want to juggle two markup syntaxes.

# Linear and tasks

Do not put references to linear issues into code. It's okay if they show up in `internal-docs`, but linear is a private, internal task-tracking system and isn't visible to anyone else, so task references are meaningless and useless.
